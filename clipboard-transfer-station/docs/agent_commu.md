# agent_commu.md

# 1. 文档目的
本文件旨在定义不同Agent之间的通信协议和数据结构，确保各个模块能够高效、可靠地进行数据交换和状态同步。

# 2. 接口定义
## API接口列表
- **获取剪贴板内容**
  - **请求**: `GET /api/clipboard`
  - **响应**: 
    - 200 OK: 返回当前剪贴板内容
    - 404 Not Found: 未找到剪贴板内容

- **更新剪贴板内容**
  - **请求**: `POST /api/clipboard`
  - **请求体**: 
    ```json
    {
      "content": "新的剪贴板内容"
    }
    ```
  - **响应**: 
    - 200 OK: 更新成功
    - 400 Bad Request: 请求体格式错误

## 请求和响应格式
- 所有请求和响应均采用JSON格式。
- 错误响应应包含错误代码和描述。

# 3. 状态同步方案
## 选择的方案
- **WebSocket**: 选择WebSocket作为状态同步方案，以实现实时数据更新和双向通信。

## 实现细节
- WebSocket连接将在客户端启动时建立，保持长连接以接收状态更新。
- 服务器将推送剪贴板内容的变化到所有连接的客户端。

# 4. 数据结构
## 共享内存结构
- 共享内存将包含以下字段：
  ```json
  {
    "clipboardContent": "当前剪贴板内容",
    "lastUpdated": "最后更新时间戳"
  }
  ```

## 数据格式定义
- 剪贴板内容应为字符串类型，支持多种格式（文本、图片等）。

# 5. 版本控制
## 数据结构修改的版本管理策略
- 所有数据结构的修改应在文档中记录版本号，确保各个模块能够兼容。
- 每次修改后，需更新版本号并记录变更日志。

# 6. 其他注意事项
